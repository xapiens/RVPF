Overview

The RVPF-JNLP child project provides an implementation of a subset of JNLP to support secure dynamic update of distributed RVPF services.

Context

The reference JNLP implementation (Java Web Start) expects to be executed in an interactive GUI environment which is not an appropriate requirement for applications running as background services.

The RVPF JNLP implementation allows such applications but has many limitations:

  • Offline mode is always enabled.
  • No sandbox environment.
  • Resources must always be signed.
  • Update check is done only at launch time, with a timeout.
  • JRE update is not supported.
  • Only the 'application-desc' application descriptor is supported.
  • The application always runs inside the same VM.
  • Download is always 'eager'.
  • Version-base download is not supported.
  • Extension resources are not supported.
